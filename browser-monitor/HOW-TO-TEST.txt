MENU CHANGE MONITOR - QUICK TESTING GUIDE
=================================================================

Follow these steps to test the extension immediately:

STEP 1: INSTALL THE EXTENSION
------------------------------
1. Open Google Chrome
2. Go to: chrome://extensions/
3. Enable "Developer mode" (toggle in top right)
4. Click "Load unpacked"
5. Select this folder: /Users/mattdavis/Documents/browser-monitor
6. Extension should now appear in your toolbar


STEP 2: GET CLAUDE API KEY
---------------------------
1. Go to: https://console.anthropic.com/
2. Sign up (free tier available)
3. Navigate to "API Keys" section
4. Click "Create Key"
5. Copy the key (starts with sk-ant-)


STEP 3: CONFIGURE EXTENSION
----------------------------
1. Click the Menu Monitor icon in Chrome toolbar
2. Paste your Claude API key
3. Enter test URL: https://test-allergy-site.com
4. Click "Save Settings"
5. You should see "Settings saved!" message


STEP 4: TEST ON GODADDY-STYLE PAGE
-----------------------------------
1. Open in Chrome:
   file:///Users/mattdavis/Documents/browser-monitor/test-pages/godaddy-style-menu.html

2. Click extension icon
3. Toggle "Monitor this website" to ON
4. Click "Test Detection Now"
5. Wait 3-5 seconds for initial scan

6. Click "Add New Dish" button on page
7. Wait 3-5 seconds
8. NOTIFICATION SHOULD APPEAR showing the new dish!

9. Click "Modify Description" to add allergen
10. Wait 3-5 seconds
11. NOTIFICATION SHOULD APPEAR warning about allergen change!


STEP 5: TEST ON WIX-STYLE PAGE
-------------------------------
1. Open in Chrome:
   file:///Users/mattdavis/Documents/browser-monitor/test-pages/wix-style-menu.html

2. Enable monitoring (toggle ON)
3. Click "Test Detection Now"
4. Use test controls to make changes
5. Verify notifications appear


STEP 6: TEST ON WORDPRESS-STYLE PAGE
-------------------------------------
1. Open in Chrome:
   file:///Users/mattdavis/Documents/browser-monitor/test-pages/wordpress-style-menu.html

2. Enable monitoring
3. Click "Test Detection Now"
4. Use test controls to make changes
5. Verify notifications appear


WHAT TO VERIFY:
---------------
✓ Notification appears within 5 seconds of change
✓ Shows correct change details (added/removed/modified)
✓ "Update Allergy Information" button works
✓ Can dismiss notification
✓ Works across all three test pages
✓ Change history visible in extension popup


TROUBLESHOOTING:
----------------
- No notification? Wait 5 seconds (debounce delay)
- First scan doesn't show changes (needs baseline)
- Check browser console (F12) for errors
- Verify API key is correct
- Make sure monitoring toggle is ON


EXPECTED BEHAVIOR:
------------------
1. First visit: Extension scans and saves baseline
2. Make change: Extension detects and analyzes
3. Notification: Popup shows what changed
4. Click button: Opens allergy website
5. New baseline: Current state becomes new baseline


SUCCESS CRITERIA:
-----------------
✓ All three test pages detect changes
✓ Notifications are clear and actionable
✓ No errors in console
✓ Extension survives page refresh
✓ Settings persist across browser restarts
